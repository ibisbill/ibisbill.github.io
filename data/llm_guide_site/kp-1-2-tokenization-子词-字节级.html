1.2 Tokenization（子词/字节级）

手册要点 + 例题答案（原文整理）

概述：Tokenizer 把文本映射为离散符号序列，决定了训练目标的粒度、序列长度与跨语言鲁棒性。

你需要掌握：

  • 子词方法：BPE、WordPiece、Unigram LM（SentencePiece）。
  • Byte-level：以字节为基础，减少 OOV，适应噪声与多语言。
  • 词表大小：压缩率、速度、泛化之间的权衡；特殊 token 设计。

工程提示：

  • Tokenizer 训练要有代表性语料采样；否则会对特定域极度不友好。
  • 对中文：子词往往不是“字”为单位，需要关注 token 长度分布。

常见误区：

  • 用 Perplexity 比较不同 tokenizer 的模型优劣（不公平）；建议补充 bits-per-byte 或任务指标。

Transformer 与 LLM 学习手册（含例题与答案）

Q1：BPE 的核心迭代步骤是什么？
A1：统计相邻符号对的频次，反复合并最频繁的对，直到达到词表大小或停止条件。
Q2：为什么 byte-level tokenization 更鲁棒？
A2：任何文本都可表示为字节序列，避免 OOV；对噪声、emoji、混合语言更稳定。

公式 / 代码 / 交互补充（重点）

Tokenization：BPE / SentencePiece / Byte-level

Tokenizer 决定了“一个 token 是什么”，进而影响长度、成本、以及 PPL 的可比性。

代码解读要点（读实现时看这些）

  • 归一化：NFKC、大小写、空白、特殊符号处理（对中英文混排尤其关键）
  • 预分词：按空白/标点切分或 byte-level 直接从字节开始
  • 合并规则：BPE merge table / unigram LM 的概率模型
  • 特殊 token：BOS/EOS/PAD/UNK，以及对话模板中的 role 标记

原论文与代码库

  • BPE：Sennrich et al., 2016 Neural Machine Translation of Rare Words with Subword Units
  • SentencePiece：Kudo & Richardson, 2018 SentencePiece
  • 字节级 BPE（GPT-2）：Radford et al., 2019（见 GPT-2 tokenizer 设计）
  • 实现：huggingface/tokenizers、google/sentencepiece
