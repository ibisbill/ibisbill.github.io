4.1 数据来源谱系与覆盖

数据是 LLM 的燃料。了解数据的来源、分布及其对模型能力的影响。

模块 4 返回模块页

核心知识点详细讲解

1. 数据来源谱系 (Data Pedigree)

现代大语言模型（如 GPT-4, Llama-3）的训练数据通常包含数万亿个 token，涵盖了人类知识的方方面面。主要来源包括：

  • Web 网页: 如 Common Crawl，提供极高的规模与多样性，但噪声大。
  • 代码 (Code): 如 GitHub，是逻辑推理能力的关键来源。代码的严格语法和可验证性对模型很有帮助。
  • 书籍 (Books): 如 BookCorpus，提供长文本建模能力、丰富的叙事和深度的知识。
  • 百科 (Wikipedia): 高质量、事实性强的权威数据。
  • 学术论文: 如 ArXiv, PubMed，涵盖科学前沿知识。
  • 数学数据: 包含公式、逻辑推导过程。

2. 数据来源对模型能力的影响

graph TD Data[训练数据] --> Web[Web 网页] Data --> Code[代码] Data --> Academic[学术/书籍] Web --> Knowledge[通用知识 & 多样性] Code --> Reasoning[逻辑推理 & 结构化] Academic --> Professional[专业深度 & 长文本]

3. 私域数据与合规性

在企业级应用中，往往需要加入私域数据（如内部文档、工单、FAQ）。

  • PII (个人身份信息) 脱敏: 必须剔除姓名、地址、电话等敏感信息。
  • 权限审计: 确保模型训练使用的文档不违反企业保密协议。
  • 合规记录: 建立数据台账，记录数据的来源、处理时间、清洗规则等，以便审计。

工程要点总结

  • 数据分桶 (Bucketing): 将数据按领域（代码、文学、科学等）和质量分桶，便于后续动态调整采样权重。
  • 多语种配比: 英语通常占主导，但加入适量其他语种（如中文、德语）能显著提升模型的跨语言泛化能力。
  • 质量高于数量: 随着研究深入，业界发现使用更高质量的小规模数据（如经过精选的教科书数据）训练出的模型，性能可能优于使用低质量海量数据训练出的模型（如 Phi 系列）。

例题与答案：

Q1：为什么代码数据会显著影响模型的推理/结构化能力？
A1：代码包含严格的逻辑结构、复杂的嵌套关系以及明确的输入输出反馈。学习代码有助于模型理解长距离依赖和执行逻辑，从而提升非代码任务（如复杂推理）的表现。
Q2：私域数据加入预训练前必须先做什么？
A2：必须进行严格的合规性评估（PII 脱敏、版权确认、密级审查），并建立完整的数据处理流水线记录。
