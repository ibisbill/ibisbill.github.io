核心知识点详细讲解

1. Unicode 规范化 (Unicode Normalization)

相同的文字在 Unicode 中可能有多种表示。例如 “é” 可以表示为一个单一字符（\u00E9），也可以表示为 “e” 加上重音符号（\u0065\u0301）。

  • NFC (Normalization Form C): 先分解再组合，最常用的形式。
  • NFD (Normalization Form D): 完全分解。
  • NFKC / NFKD: 兼容性规范化。会将带有格式的字符（如上标 ²）折叠为标准字符（如 2）。

2. 空白与不可见字符

空白字符不仅是空格。处理不当会导致模型在推理时出现奇怪的 token 边界。

  • 常见空白: \n, \r, \t, 以及全角空格 \u3000。
  • 不可见字符: 零宽空格 (ZWSP, \u200B) 等。在安全领域，攻击者常利用这些字符绕过关键词过滤或进行 Prompt 注入。

3. 语言识别与脚本清洗

  • 语言检测: 使用 fastText 等工具识别文本语种，用于后续的数据配比。
  • 脚本处理: 统一处理中英文混排中的标点符号（如全角逗号转半角，或反之）。
  • 特殊格式: 对代码段、公式（LaTeX）、表格（Markdown）等需要保留结构的文本，通常需要特定的正则表达式保护。

代码示例：Python 中的 Unicode 处理

import unicodedata

# 1. NFC 示例
s1 = 'e\u0301' # e + '
s2 = '\u00e9'  # é
print(f"Original equal: {s1 == s2}")
print(f"NFC equal: {unicodedata.normalize('NFC', s1) == unicodedata.normalize('NFC', s2)}")

# 2. NFKC 折叠示例
s3 = '②'
print(f"NFKC result: {unicodedata.normalize('NFKC', s3)}") # 输出 '2'

# 3. 清除不可见字符
import re
text = "Hello\u200BWorld" # 含有零宽空格
clean_text = re.sub(r'[\u200b-\u200f\uFEFF]', '', text)
print(f"Cleaned: {clean_text}")

手册要点 + 例题答案

概述：输入文本的“细节”会显著影响 tokenization、训练分布与安全边界。规范化的目标是降低无意义差异、控制噪声与风险。

工程提示：

  • 做安全与对齐时必须考虑：不可见字符可用于 prompt 注入与绕过过滤。
  • 对话数据要统一模板，否则模型学习到“格式偏差”。

例题与答案：

Q1：为什么需要处理不可见字符（如零宽空格）？
A1：它们可能改变 token 序列或绕过规则/过滤器，造成注入与安全风险。
Q2：NFKC 规范化的潜在风险是什么？
A2：会把某些字符折叠为“等价形式”，可能改变代码/公式的精确语义，需要按场景选择。
